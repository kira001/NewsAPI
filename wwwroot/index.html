C# Backend homework

Consegna

L’ obiettivo di questo test è di realizzare un servizio API Rest per la gestione di notizie da visualizzare in un frontend contenuti come da esempio.

Requisiti

  • 1) API non autenticata che restituisce l’elenco delle notizie da visualizzare ed il dettaglio
  • 2) API di autenticazione usando token Bearer come metodo di autenticazione
  • 3) API autenticata per la creazione, modifica, cancellazione delle notizie

    • Vincoli:

  • Utilizzo di un database Relazionale
  • Utilizzo di .NET Framework > 4.6
  • Utilizzo di Entity Framework come ORM

    • Informazioni per l'utilizzo

    L'API è stata sviluppata seguendo le specifiche primarie della consegna, in particolare viene utilizzato un database relazionare per la gestione delle news e il C# come linguaggio principale.

    Come procedere con l'esecuzione delle operazioni

    Per utilizzare correttamente l'API e le sue funzionalità bisogna autentificarsi, per far cio si deve eseguire la chiamata POST dedicata alla generazione del TOKEN e inserire i due parametri ("admin", "admin1234") nei campi che vengono richiesti al momento della sua esecuzione, una volta che viene generato il token bisognerà incollarlo nella sezione authorize e successivamente si potranno eseguire le operazioni desiderate.
